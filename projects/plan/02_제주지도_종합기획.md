## 1. 제주 실시간 날씨 레이어

### 1.1 개요

제주도는 동서남북 기후 편차가 크고, 산간/해안 기온차가 5-10도에 달한다. 기상청 AWS(자동기상관측장비) 28개소 데이터를 실시간으로 지도 위에 시각화하고, 날씨 조건에 따라 자동으로 코스를 전환 추천한다.

### 1.2 데이터 소스

| 소스 | 갱신주기 | 항목 | API |
|------|----------|------|-----|
| 기상청 초단기실황 | 매시 정각 | 기온, 강수량, 풍속, 습도 | 공공데이터포털 REST |
| 기상청 동네예보 | 3시간 | 3일 예보, 강수확률 | 공공데이터포털 REST |
| 제주 AWS 28개소 | 1분 | 기온, 풍향풍속, 강수 | 기상자료개방포털 |
| 해양기상부이 | 10분 | 파고, 수온, 풍속 | 기상청 해양기상 API |
| 제주관광공사 CCTV | 실시간 | 주요 관광지 영상 | 제주관광정보센터 |
| UV지수/미세먼지 | 1시간 | 자외선, PM2.5, PM10 | 에어코리아 API |

### 1.4 날씨별 자동 코스 추천 로직

| 날씨 조건 | 자동 전환 | 추천 카테고리 | 예시 장소 |
|-----------|----------|--------------|----------|
| 맑음 + 기온 20-28° | 실외 우선 | 올레길, 뷰포인트, 해변 | 월정리해변, 사려니숲길 |
| 맑음 + 기온 10° 미만 | 실외(방한) | 등산, 러닝, 온천 | 한라산 영실코스, 중문온천 |
| 비 (강수 1-10mm/h) | 실내 전환 | 카페, 박물관, 체험 | 아쿠아플라넷, 유리의성 |
| 폭우 (강수 10mm+ ) | 완전 실내 | 맛집, 실내체험, 숙소근처 | 흑돼지거리, 제주맥주양조장 |
| 강풍 (풍속 14m/s+) | 산간 회피 | 해안 저지대, 실내 | 이호테우해변 회피 → 실내카페 |
| 파고 2m+ | 해양 금지 | 서핑/다이빙 취소 알림 | 대체: 실내 서핑풀 |
| 안개 (시정 1km 미하) | 산간 회피 | 해안 드라이브, 카페 | 1100고지 회피 → 해안도로 |
| UV지수 8+ | 그늘 코스 | 숲길, 동굴, 실내 | 비자림, 만장굴 |

### 1.5 자동 전환 알고리즘

```
function autoSwitchCourse(currentCourse, weather) {
  // 위험도 점수 산출
  riskScore = calcRisk(weather.rain, weather.wind, weather.visibility)
  
  if (riskScore >= 80) {
    // 긴급 전환: 완전 실내 코스로 교체
    return findIndoorAlternative(currentCourse, radius=5km)
  } else if (riskScore >= 50) {
    // 부분 전환: 실외 스팟만 실내로 교체
    return replaceOutdoorSpots(currentCourse, weather)
  } else if (riskScore >= 30) {
    // 경고만 표시, 우비/방풍 준비물 안내
    return addWeatherWarning(currentCourse, weather)
  }
  return currentCourse  // 변경 없음
}
```

## 2. 제주 스케줄 빌더

### 2.1 개요

여행 일정을 지도 위에서 드래그앤드롭으로 구성하고, 동행자와 실시간 공동편집이 가능한 캔버스 기반 스케줄러. Figma/Notion의 협업 UX를 여행 일정에 적용한다.

### 2.2 핵심 기능 매트릭스

| 기능 | 설명 | 기술 스택 | 우선순위 |
|------|------|----------|---------|
| 캔버스 편집 | 무한 캔버스 위에 장소 카드 배치 | Liveblocks + Zustand | P0 |
| 드래그앤드롭 타임라인 | 시간축 위로 장소를 끌어 배치 | dnd-kit + framer-motion | P0 |
| 실시간 공동편집 | 커서 공유, 동시 수정, 충돌 해결 | Liveblocks CRDT | P0 |
| 동선 자동계산 | 장소 간 이동시간/거리 자동 산출 | Kakao Mobility API | P0 |
| 비용 정산 | 예상비용 합산, 인당 정산, 결제 연동 | 자체 정산 엔진 | P1 |
| 최적 동선 | TSP 알고리즘 기반 경로 최적화 | OR-Tools / 자체 구현 | P1 |
| 일정 템플릿 | 인기 일정 복제, 커스터마이징 | MongoDB Template 컬렉션 | P2 |
| 오프라인 모드 | 일정 로컬 저장, 오프라인 열람 | Service Worker + IndexedDB | P2 |

### 2.5 동선 자동계산 상세

| 이동수단 | 평균속도 | API | 비고 |
|----------|---------|-----|------|
| 자가용/렌트카 | 실시간 | Kakao Mobility 길찾기 | 실시간 교통 반영 |
| 대중교통 | 실시간 | 제주버스정보시스템 | 급행/일반/마을버스 구분 |
| 도보 | 4km/h | Kakao 도보 | 올레길 등 보행전용 |
| 자전거 | 15km/h | 자체 계산 | 자전거도로 우선 |
| 택시 | 실시간 | Kakao T API | 예상 요금 포함 |

**동선 최적화 알고리즘**:
- 입력: N개 장소, 출발지(숙소), 각 장소 희망 체류시간
- 제약: 영업시간, 식사시간대, 이동시간
- 목적: 총 이동시간 최소화 (TSP 변형)
- 해법: N < 12 → 완전탐색, N >= 12 → 유전 알고리즘 (100세대, 인구 50)

## 3. 제주 소셜 허브

### 3.1 개요

제주 여행자와 거주민을 연결하는 취미 기반 커뮤니티. 같은 시기 제주를 방문하는 사람들끼리 취미(서핑, 등산, 러닝, 낚시 등)를 매칭하고, 로컬 이벤트/모임을 지도 위에 표시한다.

### 3.2 사용자 세그먼트

| 세그먼트 | 비율(추정) | 특성 | 핵심 니즈 |
|----------|-----------|------|----------|
| 단기 여행자 | 55% | 2-4일 체류, 커플/가족/친구 | 현지인 추천, 특별 경험 |
| 장기 체류자 | 15% | 1주-3개월, 원격근무/한달살기 | 커뮤니티, 루틴 장소 |
| 제주 거주민 | 20% | 영주권자/이주민 | 새로운 모임, 로컬 이벤트 |
| 리피터 | 10% | 연 3회+ 방문 | 새로운 코스, 숨은 명소 |

### 3.3 취미 기반 모임 카테고리

| 카테고리 | 아이콘 | 최소인원 | 모임 형태 | 예시 |
|----------|--------|---------|----------|------|
| 서핑 | 🏄 | 2명 | 함께타기/레슨 공유 | "4/15 중문 서핑 같이 하실 분" |
| 등산/올레 | 🥾 | 3명 | 함께 걷기 | "한라산 영실코스 새벽등반" |
| 러닝 | 🏃 | 2명 | 함께 달리기 | "제주 해안도로 10km 러닝" |
| 낚시 | 🎣 | 2명 | 포인트 공유/동행 | "차귀도 갯바위 낚시 동행" |
| 다이빙 | 🤿 | 2명 | 버디/펀다이빙 | "서귀포 문섬 펀다이빙 버디 구함" |
| 자전거 | 🚴 | 2명 | 라이딩 그룹 | "동부해안 100km 라이딩" |
| 사진 | 📷 | 2명 | 출사/스팟공유 | "성산일출봉 일출 출사 모임" |
| 맛집탐방 | 🍜 | 2명 | 함께 식사 | "제주 흑돼지 맛집 투어" |
| 카페투어 | ☕ | 2명 | 카페 탐방 | "애월 카페거리 투어" |
| 요가/명상 | 🧘 | 3명 | 야외 요가 | "함덕해변 선라이즈 요가" |

### 3.4 모임 매칭 알고리즘

```
매칭 점수 = (취미 일치도 × 0.35)
          + (일정 겹침도 × 0.30)
          + (위치 근접도 × 0.15)
          + (연령대 유사도 × 0.10)
          + (평판 점수 × 0.10)

취미 일치도: 동일 카테고리 = 1.0, 유사 카테고리 = 0.5
일정 겹침도: 겹치는 날/전체 체류일
위치 근접도: 숙소 간 거리 기반 (10km 이내 = 1.0)
연령대: 같은 연령대 = 1.0, ±10세 = 0.7, ±20세 = 0.3
평판: 이전 모임 후기 평점 평균 (최초 = 0.7)
```

### 3.5 이벤트 시스템

| 이벤트 유형 | 주최자 | 지도 표시 | 참가비 |

## 4. AI 제주 코스 메이커

### 4.1 개요

사용자가 자연어로 취향을 입력하면, AI가 최적의 제주 여행 코스를 자동 생성한다. 일정, 동선, 비용, 날씨까지 모두 고려한 맞춤형 코스를 30초 이내에 제공한다.

```
|  └────────────────────────────────────────────────────────────┘   |

|                                                                    |
```

### 4.3 AI 파이프라인

| 단계 | 처리 내용 | 모델/기술 | 소요시간 |
|------|----------|----------|---------|
| 1. 의도 파싱 | 자연어 → 구조화된 취향 데이터 | Claude 3.5 Sonnet | ~2초 |
| 2. 후보 생성 | 조건 매칭으로 장소 후보 50-100개 추출 | PostGIS 공간쿼리 + 필터 | ~1초 |
| 3. 날씨 반영 | 여행일 날씨 예보로 실외/실내 조절 | 날씨 레이어 연동 | ~0.5초 |
| 4. 코스 조합 | 동선/시간/비용 최적화된 코스 3개 생성 | 자체 최적화 엔진 | ~3초 |
| 5. 검증 | 영업시간, 휴무일, 예약 필요 여부 확인 | 실시간 데이터 검증 | ~1초 |
| 6. 프레젠테이션 | 지도 + 타임라인 + 비용 요약 생성 | 프론트엔드 렌더 | ~1초 |

**총 소요시간: 약 8-10초** (로딩 애니메이션으로 체감 시간 단축)

### 4.4 취향 벡터 모델

```
취향 벡터 (12차원):

  culture:    0.0 ~ 1.0,   // 문화/역사 선호도

  photo:      0.0 ~ 1.0,   // 포토스팟 선호도

  pace:       0.0 ~ 1.0,   // 일정 밀도 (0=여유, 1=빡빡)
```

| 클러스터 | Supercluster | 핀 클러스터링 | WASM 기반, 60fps |
| 경로 | Turf.js | 지오 연산 (버퍼, 교차) | 클라이언트 사이드 연산 |
| 오프라인 | Mapbox Offline | 타일 다운로드/캐싱 | 제주도 전체 약 150MB |

**Mapbox + Kakao 하이브리드 전략:**
- Mapbox: 베이스맵 렌더링, 스타일 커스터마이징, 3D 지형, 코스 시각화
- Kakao: 한국어 POI 검색, 상세 주소, 길찾기 API, 로드뷰
- 전환: 사용자 액션에 따라 레이어 단위로 Kakao 데이터를 Mapbox 위에 오버레이

#### 5.6.3 PostGIS 공간 쿼리 최적화

```sql
-- 1. 반경 내 POI 검색 (인덱스 활용)
SELECT id, name, category, rating,
       ST_Distance(location::geography, 
                   ST_MakePoint(126.5312, 33.4996)::geography) AS dist_m
FROM spots
WHERE ST_DWithin(location::geography,
                 ST_MakePoint(126.5312, 33.4996)::geography,
                 5000)  -- 5km 반경
  AND category = 'cafe'
ORDER BY rating DESC
LIMIT 20;

-- 2. 바운딩 박스 내 POI (뷰포트 기반 로딩)
SELECT id, name, category, 
       ST_AsGeoJSON(location) as geojson
FROM spots
WHERE location && ST_MakeEnvelope(126.1, 33.1, 126.9, 33.6, 4326)
  AND category = ANY(ARRAY['cafe','restaurant'])
ORDER BY rating DESC
LIMIT 100;

-- 3. 코스 근처 POI (코스 따라가기 중 주변 추천)
SELECT s.id, s.name, s.category,
       ST_Distance(s.location::geography, c.route::geography) AS dist_m
FROM spots s, courses c
WHERE c.id = 'course-uuid-here'
  AND ST_DWithin(s.location::geography, c.route::geography, 500)
  AND s.category IN ('cafe', 'toilet', 'convenience')
ORDER BY ST_LineLocatePoint(c.route, s.location), dist_m;

-- 4. 클러스터 집계 (줌 레벨 10)
SELECT ST_ClusterKMeans(location, 20) OVER() AS cluster_id,
       category, COUNT(*) as cnt,
       ST_Centroid(ST_Collect(location)) as center
FROM spots
WHERE location && ST_MakeEnvelope(126.1, 33.1, 126.9, 33.6, 4326)
GROUP BY cluster_id, category;
```

#### 5.6.4 오프라인 지원

| 항목 | 구현 | 용량 |
|------|------|------|
| 지도 타일 | Mapbox Offline 패키지 (제주도 전체 줌 8-16) | ~150MB |
| POI 데이터 | IndexedDB에 즐겨찾기/일정 장소 캐싱 | ~5MB |
| 코스 데이터 | 선택한 코스의 GPX + 고도 데이터 | ~2MB/코스 |
| 이미지 | Service Worker 캐싱 (최근 본 장소 사진) | ~50MB |
| 일정 데이터 | LocalStorage + IndexedDB 동기화 | ~1MB |
| 총 오프라인 패키지 | 제주도 완전 오프라인 모드 | ~210MB |

**오프라인 동기화 전략:**
1. 여행 전: WiFi 환경에서 일정 관련 데이터 일괄 다운로드
2. 여행 중: 변경사항 로컬 저장, 네트워크 복귀 시 자동 동기화
3. 충돌 해결: Last-Write-Wins + 수동 머지 UI 제공
