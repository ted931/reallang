## 4. AI 제주 코스 메이커

### 4.1 개요

사용자가 자연어로 취향을 입력하면, AI가 최적의 제주 여행 코스를 자동 생성한다. 일정, 동선, 비용, 날씨까지 모두 고려한 맞춤형 코스를 30초 이내에 제공한다.

### 4.2 입력 인터페이스

```
+------------------------------------------------------------------+
|  [AI 코스 메이커]  ✨                                             |
+------------------------------------------------------------------+
|                                                                    |
|  "어떤 제주 여행을 원하시나요?"                                    |
|                                                                    |
|  ┌────────────────────────────────────────────────────────────┐   |
|  │  "커플이고요, 2박3일인데 바다 좋아하고 맛있는 거 먹고      │   |
|  │   싶어요. 예산은 50만원 정도? 렌트카 있어요"               │   |
|  └────────────────────────────────────────────────────────────┘   |
|                                                    [코스 만들기 →] |
|                                                                    |
|  또는 빠른 선택:                                                   |
|  ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐                     |
|  │커플 │ │가족 │ │친구 │ │혼자 │ │효도 │ │단체 │                     |
|  └────┘ └────┘ └────┘ └────┘ └────┘ └────┘                     |
|                                                                    |
|  ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐                             |
|  │액티 │ │힐링 │ │맛집 │ │문화 │ │자연 │                             |
|  │비티 │ │여행 │ │투어 │ │탐방 │ │탐험 │                             |
|  └────┘ └────┘ └────┘ └────┘ └────┘                             |
+------------------------------------------------------------------+
```

### 4.3 AI 파이프라인

| 단계 | 처리 내용 | 모델/기술 | 소요시간 |
|------|----------|----------|---------|
| 1. 의도 파싱 | 자연어 → 구조화된 취향 데이터 | Claude 3.5 Sonnet | ~2초 |
| 2. 후보 생성 | 조건 매칭으로 장소 후보 50-100개 추출 | PostGIS 공간쿼리 + 필터 | ~1초 |
| 3. 날씨 반영 | 여행일 날씨 예보로 실외/실내 조절 | 날씨 레이어 연동 | ~0.5초 |
| 4. 코스 조합 | 동선/시간/비용 최적화된 코스 3개 생성 | 자체 최적화 엔진 | ~3초 |
| 5. 검증 | 영업시간, 휴무일, 예약 필요 여부 확인 | 실시간 데이터 검증 | ~1초 |
| 6. 프레젠테이션 | 지도 + 타임라인 + 비용 요약 생성 | 프론트엔드 렌더 | ~1초 |

**총 소요시간: 약 8-10초** (로딩 애니메이션으로 체감 시간 단축)

### 4.4 취향 벡터 모델

```
취향 벡터 (12차원):
{
  nature:     0.0 ~ 1.0,   // 자연 선호도
  food:       0.0 ~ 1.0,   // 미식 선호도
  activity:   0.0 ~ 1.0,   // 액티비티 선호도
  culture:    0.0 ~ 1.0,   // 문화/역사 선호도
  relaxation: 0.0 ~ 1.0,   // 휴식/힐링 선호도
  photo:      0.0 ~ 1.0,   // 포토스팟 선호도
  budget:     0.0 ~ 1.0,   // 예산 수준 (0=절약, 1=럭셔리)
  pace:       0.0 ~ 1.0,   // 일정 밀도 (0=여유, 1=빡빡)
  popularity: 0.0 ~ 1.0,   // 유명관광지 vs 숨은명소
  local:      0.0 ~ 1.0,   // 현지인 감성 선호도
  adventure:  0.0 ~ 1.0,   // 모험 선호도
  social:     0.0 ~ 1.0    // 사교 선호도
}

예시 - "커플 바다 맛집 50만원":
{
  nature: 0.8, food: 0.9, activity: 0.3,
  culture: 0.2, relaxation: 0.6, photo: 0.7,
  budget: 0.5, pace: 0.5, popularity: 0.6,
  local: 0.5, adventure: 0.3, social: 0.3
}
```

### 4.6 학습 및 개선

| 데이터 | 활용 | 수집 방법 |
|--------|------|----------|
| 코스 선택률 | A/B/C 중 어떤 코스가 많이 선택되는지 | 클릭 로그 |
| 코스 수정 내역 | AI 코스에서 어떤 장소를 빼고 넣는지 | 편집 로그 |
| 실제 방문 여부 | GPS로 실제 방문 확인 | 위치 데이터 (옵트인) |
| 만족도 평가 | 여행 후 코스 평점 | 인앱 서베이 |
| 리뷰 텍스트 | 감성 분석으로 장소 선호도 업데이트 | NLP 분석 |
