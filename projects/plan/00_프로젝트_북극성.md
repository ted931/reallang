# 제주패스 에코시스템 — 북극성 문서

> 이 문서는 "왜 이걸 만드는가"를 설명합니다.  
> 기능 명세나 유저 플로우가 아닌, 전략적 목적과 기대 결과를 담습니다.

---

## 한 줄 정의

> **제주를 여행하는 사람과 제주에서 장사하는 사람 사이에 존재하는 모든 거래를 연결하는 SEO 기반 플랫폼**

---

## 왜 만드는가

### 문제

제주에는 좋은 가게, 좋은 체험, 좋은 숙소가 많은데 — 여행자는 네이버 블로그 정보에만 의존하고, 사장님은 배민·카카오·네이버에 수수료를 내면서 홍보한다.

- 여행자: 정보가 산재되어 있고, 예약·결제·동행 모두 따로 해야 한다
- 사장님: 마케팅 비용 대비 효과를 측정하기 어렵고, 단체 손님을 끌기 더 어렵다
- 플랫폼: 네이버·카카오가 독점한 검색 시장에서 외부 웹사이트가 얻을 수 있는 클릭은 최대 8%

### 기회

이 8% 안에서 싸우는 게 아니라 — **직접 트래픽을 만든다**.

- 파티(같이 할 사람 모집) → SNS 자연 공유
- AI 일정 생성 → 검색 없이 앱에서 해결
- 가게 홍보 도구 → 사장님이 직접 콘텐츠 생산 → 외부 유입

---

## 생태계 구조

```
여행자 (수요)
    │
    ▼
[jeju-party] ←─── 파티 참가 / AI 일정 / 렌터카 동승
    │
    ├──→ [jeju-map]        지도에서 가게 발견
    │
    ├──→ [jejupass-promo]  가게 상세 / 리뷰 / 예약
    │
    └──→ [jeju-biz]        사업자용 파티 모집 / 정산
    
사장님 (공급)
    │
    ▼
[jejupass-promo] ─── 무료 홍보 등록 → AI SNS 콘텐츠 자동 생성
    │
    └──→ [jeju-biz]  파티 개설 → 단체 손님 유치 → 수수료 수익
```

**핵심 루프:**
1. 여행자가 파티에 참가 → SNS에 공유
2. 공유 → 신규 여행자 유입
3. 신규 여행자 → 가게 방문 → 사장님 매출 증가
4. 매출 증가 → 사장님 유료 플랜 전환
5. 유료 전환 → 플랫폼 수익 → 더 나은 기능 투자

## SEO 전략

### 왜 SEO인가

- 광고비 없이 지속 가능한 트래픽
- 한번 상위 노출되면 복리처럼 유입이 쌓임
- 경쟁사(네이버 플레이스)가 못 가는 영역(구글 롱테일)이 있음

### 타겟 키워드 구조

```
대형 키워드 (경쟁 극상, 당장 불가)
  제주 렌터카, 제주 맛집, 제주 여행코스

중형 키워드 (6~12개월 목표)
  제주 한달살기, 제주 서핑 초보, 제주 애견동반 카페

롱테일 키워드 (즉시 가능, 트래픽 적지만 전환율 높음)
  제주 3박4일 커플 코스 2026
  제주 한달살기 렌터카 같이 쓸 사람
  제주 서핑 초보 같이 배울 사람
```

### 콘텐츠 생산 전략

플랫폼이 직접 콘텐츠를 생산하는 게 아니라 **사용자·사장님이 생산하게 만든다**:

| 생산자 | 콘텐츠 | SEO 효과 |
|---|---|---|
| 여행자 | 파티 후기, 리뷰 | 롱테일 키워드 자연 생성 |
| 사장님 | AI 자동 홍보글 | 가게명+카테고리+지역 키워드 |
| AI | 일정 페이지 | "제주 N일 코스" 템플릿 대량 생성 |

## 단계별 목표

### 초기 (현재 ~ 6개월): SEO + 데이터 축적

**이 단계의 핵심 질문**: "사람들이 실제로 원하는가?"

목표는 수익이 아닙니다. **검색 유입 + 행동 데이터**를 쌓는 것입니다.

| 목표 | 지표 | 왜 중요한가 |
|---|---|---|
| SEO 기반 구축 | 구글 상위 노출 키워드 50개 | 광고비 없는 지속 트래픽의 시작 |
| 콘텐츠 누적 | 파티 후기 500건, 가게 리뷰 2,000건 | 롱테일 키워드 자동 생성 |
| 사용 패턴 파악 | 어떤 파티가 많이 열리나, 어떤 가게가 클릭되나 | Phase 2 유료화 기준 설정 |
| 가게 등록 | 300개 | 콘텐츠 공급 확보 |
| 파티 개설 | 월 100개 | 수요 검증 |

**이 단계에서 하지 않을 것**: 광고 판매, 유료 플랜, 투자 유치 피칭  
→ 데이터 없이 팔면 설득력이 없다.

---

### 중기 (6~18개월): 수익화 검증

SEO로 쌓인 트래픽을 처음으로 수익으로 전환합니다.

| 목표 | 지표 |
|---|---|
| 월 순방문자 | 5만 → 20만 |
| 유료 사장님 | 50명 → 200명 |
| MRR | 100만원 → 1,500만원 |
| 파티 거래액 | 월 500만원 |

**수익화 순서**:
1. 사장님 구독 (월 3~30만원) — 가장 단순, 먼저 시작
2. 렌터카 제휴 수수료 — 예약 건당 3,000~15,000원
3. 파티 예약 수수료 — 거래액의 3~5%
4. 광고 (피처드 노출) — 카테고리당 50만원/월

**이 단계의 핵심 질문**: "어떤 사장님이, 왜, 얼마에 결제하는가?"  
→ 이 패턴을 찾으면 영업이 아니라 시스템으로 전환한다.

---

### 장기 (18개월~): 제주 여행 인프라

더 이상 "홍보 플랫폼"이 아닌, **제주 여행의 필수 경유지**가 됩니다.

| 목표 | 내용 |
|---|---|
| 검색 독립성 | 네이버·구글 의존 없이 직접 트래픽 30% 이상 |
| 거래 인프라 | 렌터카·숙박·체험·파티 예약이 한 곳에서 |
| B2B 확장 | 제주 관광공사, 항공사, 카드사 제휴 |
| MRR 목표 | 5,000만원 |
| 지역 확장 | 제주 검증 후 → 강원, 부산, 오키나와 |

**장기 해자 (경쟁사가 따라오기 어려운 것)**:
- 수년간 쌓인 파티 후기·가게 리뷰 데이터 (SEO 자산)
- 호스트 평점 시스템 (다른 플랫폼으로 이전 불가)
- 사장님 AI 홍보 도구에 쌓인 가게별 최적화 데이터

## 기대 효과

### 여행자 입장
- 동행 찾기 + 일정 짜기 + 렌터카 + 가게 예약이 한 곳에서 된다
- AI가 내 스타일에 맞는 코스를 짜준다
- 같이 여행한 사람들과 후기를 남긴다

### 사장님 입장
- 가입 3분, 이후 AI가 자동으로 SNS 홍보글을 만들어준다
- 파티(단체 손님)를 직접 모집해서 비수기에도 매출을 만든다
- 광고비 없이 네이버·인스타 노출이 된다

### 플랫폼 입장
- 사용자가 만드는 콘텐츠로 SEO가 자동으로 쌓인다
- 가게가 많을수록 여행자가 오고, 여행자가 많을수록 가게가 온다 (네트워크 효과)
- 제주 여행의 모든 거래를 통과하는 인프라가 된다

## 초기 사람을 어떻게 모으는가

### 콜드 스타트 문제

플랫폼의 가장 어려운 시기 — 가게도 없고, 파티도 없고, 여행자도 없다.  
순서가 중요하다: **공급(사장님/호스트)을 먼저, 수요(여행자)를 나중에.**

### Phase 0: 씨앗 심기 (첫 3개월)

**① 서핑스쿨·낚시투어·자전거샵 10곳을 직접 섭외**
- 0% 수수료 6개월 보장 제안
- 이들이 파티를 열면 → 그 파티 참가자들이 자연스럽게 후기를 남김
- 목표: 파티 50개, 후기 200개 → 롱테일 SEO 시작

**② 인스타그램 감성 계정 운영**
- "제주에서 이런 사람 만났다" 스토리 형식
- 실제 파티 후기를 허락 받고 공유
- 팔로워 목표 3개월 5,000명

**③ 제주 관련 커뮤니티 공략**
- 에어비앤비 호스트 카페, 제주살기 네이버 카페, 당근 제주
- "파티 개설하면 6개월 무료 노출" 홍보

**④ 한달살기 유튜버/블로거 협업**
- 제주 한달살기 콘텐츠 크리에이터 5~10명 협업
- 댓가: 카페패스 무료 제공 + 파트너 뱃지
- 효과: 영상 하나당 구독자 트래픽 직접 유입

## 사람들이 쓸 수밖에 없는 이유

### 여행자 관점 — 다른 곳에서는 못 얻는 것

| 욕구 | 기존 방법 | 제주파티 |
|---|---|---|
| 동행 찾기 | 에어비앤비 체험, 당근, 오픈카톡 | 카테고리·날짜·지역으로 필터링, 1클릭 신청 |
| 혼자 여행 외로움 | 게스트하우스 라운지 눈치 | 공식적으로 "같이 놀자"가 가능 |
| 렌터카 비용 절감 | 혼자 빌림 | 동승자 모집 → 반값 |
| 일정 짜기 귀찮음 | 블로그 수십 개 읽기 | AI가 내 스타일로 1분 안에 |
| 검증된 로컬 맛집 | 네이버 리뷰 (광고 의심) | 파티 참가자 실후기 |

**핵심 차별점**: 혼자 제주 여행 온 사람이 "함께"를 원할 때 갈 곳이 없다.  
제주파티가 그 유일한 공식 창구가 된다.

### 사장님 관점 — 안 쓸 이유가 없는 구조

| 조건 | 내용 |
|---|---|
| 가입비 | 무료 |
| 설정 시간 | 3분 (네이버 스마트플레이스 정보 자동 연동) |
| 홍보 콘텐츠 | AI가 매주 자동 생성 |
| 리스크 | 없음 — 파티 수수료는 실제 매출 발생 시만 |

"무료인데 안 할 이유가 있나?" — 이게 온보딩 설득의 전부.  
이후 매출이 발생하면 유료로 자연스럽게 전환.

### 이탈 방지 — 한번 쓰면 계속 쓰게 만드는 것

| 장치 | 설명 |
|---|---|
| 파티 히스토리 | 내가 참가한 파티, 만난 사람들이 기록으로 남음 |
| 호스트 평점 | 쌓인 평점은 다른 플랫폼으로 이전 불가 |
| 단골 가게 | 자주 간 가게 저장 → 할인 쿠폰 자동 제공 |
| 카페패스 히스토리 | 방문 도장 → 시각적 달성감 |

## 우선순위 원칙

기능을 추가할 때 이 기준으로 판단합니다:

1. **SEO에 기여하는가?** — 새 URL이 생기거나, 사용자가 콘텐츠를 만드는가
2. **네트워크 효과가 있는가?** — 사용자가 늘수록 서비스가 더 좋아지는가
3. **사장님 매출에 직결되는가?** — 사장님이 돈을 벌어야 플랫폼도 돈을 번다
4. **바이럴 가능한가?** — 자연스럽게 공유하고 싶어지는가
