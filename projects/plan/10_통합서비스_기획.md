# 통합 서비스 설계 명세서

realang.store 포털의 TOP 섹션에 배치될 3개 통합 서비스의 상세 스펙.
각 서비스는 기존 프로토타입들의 기능을 하나의 UX로 통합하고, 시너지 기능을 추가한 것.

---

## 1. 제주 여행 (jeju-travel)

### 통합 대상
| 기존 프로젝트 | 핵심 기능 |
|---|---|
| jeju-weather | 10개 지역 실시간 기상 (기온·강수·풍속) |
| jeju-map | 카카오맵 연동 13개 카테고리 장소 탐색 |
| jeju-travel-planner | 자연어 입력 → AI 일정 자동 생성 |
| jeju-course-maker | 취향·인원·예산 기반 AI 코스 3개 추천 |
| weather-drive | 날씨 기반 드라이브 코스 추천 |

### URL 구조
```
/travel/                  → 메인 (여행 플로우 시작)
/travel/weather           → 실시간 날씨 탭
/travel/course            → AI 코스 메이커 탭
/travel/plan              → AI 일정 플래너 탭
/travel/map               → 장소 탐색 지도 탭
/travel/drive             → 드라이브 코스 탭
```

### 사용자 플로우
1. `/travel/` 진입 → "제주 여행 시작하기" 히어로, 5단계 가이드 표시
2. **STEP 1 — 날씨 확인**: 지역별 날씨를 확인하고, 맑은 날/비오는 날에 따라 추천이 바뀜
3. **STEP 2 — 코스 생성**: 취향(자연/맛집/액티비티), 인원, 예산 입력 → AI가 3개 코스 생성
4. **STEP 3 — 일정 생성**: 선택한 코스를 바탕으로 일자별 일정 자동 구성 (숙소·식사 포함)
5. **STEP 4 — 지도 탐색**: 일정에 포함된 장소를 지도에서 확인, 추가 장소 검색·교체
6. **STEP 5 — 드라이브**: 일정 동선을 기반으로 드라이브 코스 자동 생성

각 탭은 독립적으로도 사용 가능하지만, 위 플로우로 연결될 때 데이터가 자동 전달됨.

### 시너지 기능 (기존에 없던 것)
- **날씨 연동 코스 추천**: 날씨 탭의 실시간 데이터가 코스 메이커에 자동 전달. 비 오는 지역은 실내 코스 위주로 재배치.
- **일정 ↔ 지도 양방향 동기화**: 플래너에서 일정 변경 시 지도 마커 자동 업데이트, 지도에서 장소 추가 시 일정에 자동 삽입.
- **통합 동선 최적화**: 코스 + 일정 + 드라이브를 합산해 전체 이동거리·시간 최적화 알고리즘 적용.
- **원클릭 공유**: 완성된 여행 플랜(날씨+코스+일정+지도)을 하나의 공유 링크로 동행자에게 전달.

## 2. 제주 렌터카 (jeju-car)

### 통합 대상
| 기존 프로젝트 | 핵심 기능 |
|---|---|
| car-pick | 인원·짐·예산·선호도 기반 AI 차종 추천 |
| smart-fuel | 제주 전역 주유소 실시간 가격 비교, 경로 최적 주유소 |

### URL 구조
```
/car/                     → 메인 (렌터카 여행 시작)
/car/pick                 → AI 차종 추천 탭
/car/fuel                 → 스마트 주유 탭
/car/cost                 → 총 비용 시뮬레이터 (신규)
```

### 사용자 플로우
1. `/car/` 진입 → "렌터카 여행, 얼마나 들까?" 히어로
2. **STEP 1 — 차종 추천**: 인원·짐 개수·예산·선호(SUV/세단/경차) 입력 → AI가 3개 차종 추천 (렌트비 포함)
3. **STEP 2 — 주유 계획**: 추천 차종의 연비 데이터를 기반으로 예상 주유 횟수·비용 자동 계산. 경로상 최저가 주유소 표시.
4. **STEP 3 — 총 비용**: 렌트비 + 주유비 + 보험료 + 톨비 등 총 비용 시뮬레이션

### 시너지 기능 (기존에 없던 것)
- **차종별 연비 연동 주유 비용 예측**: car-pick에서 선택한 차종의 공인연비를 smart-fuel로 전달. "이 차로 제주 일주하면 주유비 약 X만원" 자동 계산.
- **총 비용 시뮬레이터**: 렌트비 + 주유비 + 보험 + 주차비를 통합 비교. 차종 변경 시 전체 비용이 실시간 변동.
- **제주 여행 서비스 연동**: `/travel/`의 일정이 있으면 해당 동선 기반으로 주유 계획 자동 생성.
- **경유지 기반 최적 주유소**: 여행 일정의 경유지를 분석해 "어디서 넣는 게 가장 싼지" 자동 추천.

## 3. 제주 비즈니스 (jeju-biz)

### 통합 대상
| 기존 프로젝트 | 핵심 기능 |
|---|---|
| jejupass-promo | 자영업자 무료 홍보 페이지 생성 (사진·메뉴·위치·영업시간) |
| dashboard | 매출·예약·CS 현황 통합 대시보드 |
| cs-chatbot | 예약/취소/환불 AI 자동 응대 챗봇 |

### URL 구조
```
/biz/                     → 메인 (서비스 소개 + 가입)
/biz/promo                → 홍보 페이지 관리 탭
/biz/promo/[shopId]       → 개별 가게 홍보 페이지 (공개 URL)
/biz/dashboard            → 통합 대시보드 탭
/biz/chatbot              → AI 챗봇 설정 탭
/biz/chatbot/widget       → 임베드용 챗봇 위젯
```

### 사용자 플로우

#### A. 신규 가입 (사장님)
1. `/biz/` 진입 → "제주 사장님, 홍보부터 CS까지 무료" 히어로
2. 간편 가입 (카카오/네이버 로그인)
3. **STEP 1 — 홍보 페이지 생성**: 가게명·사진·메뉴·위치·영업시간 입력 → 즉시 공개 URL 생성
4. **STEP 2 — 챗봇 연결**: 홍보 페이지 데이터를 기반으로 AI 챗봇이 자동 학습 (메뉴 문의, 위치 안내, 영업시간 답변 등)
5. **STEP 3 — 대시보드 확인**: 홍보 페이지 조회수, 챗봇 응대 건수, 문의 유형 분석 확인

#### B. 일상 운영 (기존 사장님)
1. `/biz/dashboard` → 오늘의 주요 지표 (조회수, 문의 건수, 미응대 건수)
2. 챗봇이 처리 못한 문의 → 사장님에게 알림 → 직접 답변
3. 홍보 페이지 이벤트/메뉴 업데이트 → 챗봇 자동 재학습

### 시너지 기능 (기존에 없던 것)
- **홍보 → 챗봇 자동 학습**: 홍보 페이지에 등록한 메뉴·가격·위치·영업시간을 챗봇이 자동으로 학습. 별도 챗봇 설정 불필요.
- **챗봇 대화 → 대시보드 인사이트**: 고객 문의 패턴 분석 (가장 많이 묻는 메뉴, 예약 피크 시간대) → 대시보드에 인사이트 카드로 표시.
- **통합 고객 관리**: 홍보 페이지 방문자 → 챗봇 문의자 → 실제 방문자 퍼널 추적. 전환율 데이터 대시보드에 표시.
- **제주 여행 서비스 노출**: 홍보 페이지가 `/travel/map`의 장소 데이터로 자동 등록. 여행자가 지도에서 가게를 바로 발견.

## 공통 설계 원칙

### 데이터 공유
- 3개 통합 서비스 간 데이터 연동은 공유 상태 관리(zustand 또는 URL 파라미터)로 구현
- 예: `/travel/`에서 생성한 일정을 `/car/`가 쿼리 파라미터로 받아 주유 계획 자동 생성

### 독립 접근 가능
- 각 탭은 통합 서비스의 일부이지만 독립 URL로 직접 접근 가능
- 다른 탭의 데이터가 없어도 해당 탭 단독 사용에 문제 없음

### 점진적 마이그레이션
- 기존 프로토타입 URL(`/weather/`, `/map/` 등)은 유지하되 통합 서비스 탭으로 리다이렉트
- 아이디어 놀이터에서 기존 URL도 계속 접근 가능 (하위 호환)

### 상태 배지 규칙
- **LIVE**: 실사용 가능, 안정적 운영 중
- **BETA**: 핵심 기능 동작, 일부 미완성
- **준비중**: 설계 완료, 개발 진행 중
