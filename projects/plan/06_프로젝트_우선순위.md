## 상세 분석

### 1순위: 제주 실시간 날씨 지도 (port 3004)

**즉시 착수 가능 — 보유 API 100% 활용**

- **보유 API**: DATA_GO_KR_KEY (기상청 API), VWORLD_API_KEY (지도 타일)
- **구현 범위**: 기상청 초단기실황 + 동네예보를 V-World 지도 위에 레이어로 표시
- **왜 1순위인가**:
  - 추가 API 가입 불필요, 지금 바로 개발 시작 가능
  - 지도 플랫폼 전체의 기반 인프라 역할 (2, 4, 9순위 프로젝트가 이 위에 구축)
  - 1~2주 만에 MVP 가능한 적정 복잡도
  - 제주 여행자에게 즉각적인 실용 가치 제공

### 2순위: 제주 전용 지도 플랫폼 (port 3005)

**즉시 착수 가능 — 날씨 지도 확장**

- **보유 API**: VWORLD_API_KEY, DATA_GO_KR_KEY (관광 POI 데이터)
- **구현 범위**: 13개 카테고리 핀 (맛집, 카페, 관광지 등), 코스 시스템
- **왜 2순위인가**:
  - 날씨 지도와 동일한 지도 인프라 공유 → 개발 효율 극대화
  - 공공데이터포털의 제주관광 데이터로 POI 구축 가능
  - 이후 코스 메이커, 스케줄 빌더의 기반이 됨

### 3순위: AI 여행 플래너 (port 3006)

**즉시 착수 가능 — 매출 임팩트 최대**

- **보유 API**: OPENAI_API_KEY (GPT-4o), DATA_GO_KR_KEY (관광 데이터)
- **구현 범위**: 자연어 입력 → AI가 숙소/렌터카/식당/액티비티 패키지 구성
- **왜 3순위인가**:
  - 기획서 기준 연 매출 +8억 예상, 가장 높은 비즈니스 임팩트
  - OpenAI API + 공공데이터로 MVP 구현 가능
  - 지도 플랫폼(1, 2순위) 완성 후 통합하면 시너지 극대화

### 4순위: AI 제주 코스 메이커 (port 3007)

**즉시 착수 가능 — 여행 플래너 기술 재활용**

- **보유 API**: OPENAI_API_KEY, VWORLD_API_KEY, DATA_GO_KR_KEY
- **구현 범위**: AI가 날씨/시간/취향 기반으로 최적 코스 자동 생성, 지도 위 시각화
- **왜 4순위인가**:
  - 3순위 여행 플래너와 프롬프트/데이터 파이프라인 공유
  - 날씨 지도(1순위) 데이터와 연동하면 날씨 기반 코스 전환 가능
  - 지도 플랫폼(2순위)에 코스 레이어로 통합

### 5순위: 통합 대시보드 (port 3008)

**즉시 착수 가능 — 내부 운영 효율화**

- **보유 API**: Supabase (데이터 저장/실시간 구독)
- **구현 범위**: 매출/예약/CS 실시간 현황 대시보드
- **왜 5순위인가**:
  - 외부 API 불필요, Supabase만으로 구현 가능
  - 의사결정 속도 3일 → 실시간 개선
  - 다만 고객향이 아닌 내부 도구이므로 직접 매출 임팩트 낮음

### 6순위: AI CS 챗봇 (port 3009)

**즉시 착수 가능 — CS 비용 절감**

- **보유 API**: OPENAI_API_KEY, Supabase
- **구현 범위**: FAQ 기반 자동 응답 + GPT 폴백
- **왜 6순위인가**:
  - OpenAI API로 빠르게 구축 가능 (1~2주)
  - CS 인력 3명 절감 효과
  - 단, 제주패스 자체 FAQ/정책 데이터 정리가 선행되어야 함

### 7순위: 직원관리 ERP (port 3010)

**즉시 착수 가능 — 외부 API 불필요하지만 범위 넓음**

- **보유 API**: Supabase (DB/인증)
- **구현 범위**: 근태/시프트/성과/급여 관리
- **왜 7순위인가**:
  - 외부 API 의존성 없음 → 착수 자체는 가능
  - 하지만 기능 범위가 매우 넓어 개발 기간 3~4주 이상
  - 내부 도구이므로 고객향 프로젝트 이후 착수가 적절

### 8순위: 리뷰 엔진 (port 3011)

**추가 API 필요 — 크롤링/외부 플랫폼 연동**

- **필요 API**: 네이버 검색 API, 카카오 로컬 API, Google Places API
- **구현 범위**: 리뷰 수집/감성분석/추천
- **왜 8순위인가**:
  - 전환율 +22% 효과 기대
  - 하지만 네이버/카카오/구글 API 가입 및 크롤링 법적 검토 필요
  - OpenAI로 감성분석은 가능하나 리뷰 소스 확보가 관건

### 9순위: 제주 스케줄 빌더 (port 3012)

**추가 API/라이브러리 필요 — 높은 복잡도**

- **필요 API**: Kakao Mobility API (길찾기), Liveblocks (실시간 협업)
- **구현 범위**: 드래그앤드롭 캔버스, 실시간 공동편집, 동선 최적화
- **왜 9순위인가**:
  - Liveblocks 유료 구독 + Kakao Mobility API 신청 필요
  - CRDT 기반 실시간 협업은 기술적 난이도 높음
  - 지도 플랫폼 + 코스 메이커 완성 후 통합이 효율적

### 10순위: 제주 소셜 허브 (port 3013)

**추가 API 필요 — 가장 높은 복잡도**

- **필요 API**: 실시간 채팅 (Supabase Realtime 또는 Socket.io), 소셜 로그인 (카카오/네이버), 푸시 알림 (FCM)
- **구현 범위**: 취미 매칭, 모임 생성, 채팅, 지도 이벤트 표시
- **왜 10순위인가**:
  - 사용자 기반(critical mass)이 없으면 서비스 가치 없음
  - 다른 프로젝트로 트래픽 확보 후 론칭이 효과적
  - 실시간 채팅, 매칭 알고리즘 등 기술 복잡도 최고

## 추천 개발 스프린트

### Sprint 1 (1~2주): 지도 인프라 구축
- **제주 실시간 날씨 지도** → 지도 타일 + 기상 API 연동

### Sprint 2 (2~3주): 지도 확장 + AI 핵심
- **제주 전용 지도 플랫폼** → POI 데이터 + 카테고리 시스템
- **AI 여행 플래너** → GPT-4o + 공공데이터 RAG (병렬 개발 가능)

### Sprint 3 (2주): AI 코스 + 내부 도구
- **AI 코스 메이커** → 여행 플래너 기술 재활용
- **통합 대시보드** → Supabase 기반 내부 도구

### Sprint 4 (2~3주): CS 자동화 + 인사 관리
- **AI CS 챗봇** → FAQ 데이터 정리 후 빠르게 구축
- **직원관리 ERP** → 근태/시프트 MVP부터 시작

### Sprint 5 이후: 외부 API 확보 후
- **리뷰 엔진** → 네이버/카카오 API 승인 후
- **제주 스케줄 빌더** → Kakao Mobility + Liveblocks 세팅 후
- **제주 소셜 허브** → 충분한 사용자 기반 확보 후
