### 5.2 카테고리별 상세 데이터 스키마

**카페/맛집 공통 스키마:**
```sql
CREATE TABLE spots (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  category      VARCHAR(20) NOT NULL,
  name          VARCHAR(200) NOT NULL,
  name_en       VARCHAR(200),
  address       VARCHAR(500) NOT NULL,
  location      GEOMETRY(Point, 4326) NOT NULL,
  phone         VARCHAR(20),
  
  -- 영업 정보
  business_hours JSONB,          -- {"mon": "09:00-21:00", ...}
  holidays      VARCHAR(200),    -- "매주 화요일"
  last_order    TIME,
  
  -- 상세 정보
  price_range   INT,             -- 1(저렴)-4(고가)
  avg_price     INT,             -- 인당 평균 (원)
  menu_items    JSONB,           -- [{"name":"아메리카노","price":5000}, ...]
  tags          TEXT[],          -- ["오션뷰", "반려동물동반", "키즈"]
  parking       BOOLEAN,
  wifi          BOOLEAN,
  
  -- 평가
  rating        DECIMAL(2,1),    -- 자체 평점
  review_count  INT DEFAULT 0,
  naver_rating  DECIMAL(2,1),
  kakao_rating  DECIMAL(2,1),
  
  -- 메타
  source        VARCHAR(50),
  verified      BOOLEAN DEFAULT false,
  created_at    TIMESTAMPTZ DEFAULT NOW(),
  updated_at    TIMESTAMPTZ DEFAULT NOW(),
  
  -- 공간 인덱스
  CONSTRAINT spots_location_idx 
    USING GIST(location)
);
```

**액티비티(서핑/다이빙/낚시) 추가 스키마:**
```sql
CREATE TABLE activity_spots (
  spot_id       UUID REFERENCES spots(id),
  
  -- 조건 정보
  difficulty    VARCHAR(20),     -- 초급/중급/상급
  season        INT[],           -- [5,6,7,8,9] (적합 월)
  best_time     VARCHAR(100),    -- "만조 전후 2시간"
  
  -- 환경 조건
  min_wave      DECIMAL(2,1),    -- 서핑: 최소 파고(m)
  max_wave      DECIMAL(2,1),    -- 서핑: 최대 파고(m)
  water_temp    JSONB,           -- {"jan":12,"feb":11,...}
  visibility    JSONB,           -- 다이빙: 월별 시야(m)
  fish_species  TEXT[],          -- 낚시: 어종
  
  -- 장비
  rental_available BOOLEAN,
  rental_price  JSONB,           -- {"wetsuit":20000,"board":30000}
  
  -- 안전
  hazard_notes  TEXT,
  emergency_contact VARCHAR(20),
  lifeguard     BOOLEAN
);
```

**코스(자전거/러닝/올레길) 스키마:**
```sql
CREATE TABLE courses (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  category      VARCHAR(20) NOT NULL,
  name          VARCHAR(200) NOT NULL,
  
  -- 경로
  route         GEOMETRY(LineString, 4326) NOT NULL,
  distance_km   DECIMAL(5,1) NOT NULL,
  elevation_gain INT,            -- 누적 상승(m)
  elevation_loss INT,            -- 누적 하강(m)
  highest_point INT,             -- 최고 해발(m)
  lowest_point  INT,             -- 최저 해발(m)
  
  -- 정보
  difficulty    VARCHAR(20),     -- 쉬움/보통/어려움/전문가
  estimated_time INT,            -- 예상 소요시간(분)
  surface_type  VARCHAR(50),     -- 포장/비포장/혼합/트레일
  
  -- 고도 프로필 데이터
  elevation_profile JSONB,       -- [{dist:0,elev:10},{dist:0.5,elev:25},...]
  
  -- 편의시설
  water_points  GEOMETRY(MultiPoint, 4326),  -- 급수대 위치
  rest_areas    GEOMETRY(MultiPoint, 4326),  -- 쉼터 위치
  toilets       GEOMETRY(MultiPoint, 4326),  -- 화장실 위치
  
  -- 메타
  rating        DECIMAL(2,1),
  review_count  INT DEFAULT 0,
  photo_count   INT DEFAULT 0,
  created_at    TIMESTAMPTZ DEFAULT NOW()
);
```

#### 5.6.3 PostGIS 공간 쿼리 최적화

```sql
-- 1. 반경 내 POI 검색 (인덱스 활용)
SELECT id, name, category, rating,
       ST_Distance(location::geography, 
                   ST_MakePoint(126.5312, 33.4996)::geography) AS dist_m
FROM spots
WHERE ST_DWithin(location::geography,
                 ST_MakePoint(126.5312, 33.4996)::geography,
                 5000)  -- 5km 반경
  AND category = 'cafe'
ORDER BY rating DESC
LIMIT 20;

-- 2. 바운딩 박스 내 POI (뷰포트 기반 로딩)
SELECT id, name, category, 
       ST_AsGeoJSON(location) as geojson
FROM spots
WHERE location && ST_MakeEnvelope(126.1, 33.1, 126.9, 33.6, 4326)
  AND category = ANY(ARRAY['cafe','restaurant'])
ORDER BY rating DESC
LIMIT 100;

-- 3. 코스 근처 POI (코스 따라가기 중 주변 추천)
SELECT s.id, s.name, s.category,
       ST_Distance(s.location::geography, c.route::geography) AS dist_m
FROM spots s, courses c
WHERE c.id = 'course-uuid-here'
  AND ST_DWithin(s.location::geography, c.route::geography, 500)
  AND s.category IN ('cafe', 'toilet', 'convenience')
ORDER BY ST_LineLocatePoint(c.route, s.location), dist_m;

-- 4. 클러스터 집계 (줌 레벨 10)
SELECT ST_ClusterKMeans(location, 20) OVER() AS cluster_id,
       category, COUNT(*) as cnt,
       ST_Centroid(ST_Collect(location)) as center
FROM spots
WHERE location && ST_MakeEnvelope(126.1, 33.1, 126.9, 33.6, 4326)
GROUP BY cluster_id, category;
```

#### 5.6.4 오프라인 지원

| 항목 | 구현 | 용량 |
|------|------|------|
| 지도 타일 | Mapbox Offline 패키지 (제주도 전체 줌 8-16) | ~150MB |
| POI 데이터 | IndexedDB에 즐겨찾기/일정 장소 캐싱 | ~5MB |
| 코스 데이터 | 선택한 코스의 GPX + 고도 데이터 | ~2MB/코스 |
| 이미지 | Service Worker 캐싱 (최근 본 장소 사진) | ~50MB |
| 일정 데이터 | LocalStorage + IndexedDB 동기화 | ~1MB |
| 총 오프라인 패키지 | 제주도 완전 오프라인 모드 | ~210MB |

**오프라인 동기화 전략:**
1. 여행 전: WiFi 환경에서 일정 관련 데이터 일괄 다운로드
2. 여행 중: 변경사항 로컬 저장, 네트워크 복귀 시 자동 동기화
3. 충돌 해결: Last-Write-Wins + 수동 머지 UI 제공
