## 3. 직원관리 ERP 설계

### 3.1 현황 진단

| 영역 | 현재 방식 | 문제점 | 심각도 |
|------|----------|--------|--------|
| 근태 | 엑셀 수기 입력 | 월말 3일 소요, 오류 월 5건 | ★★★ |
| 시프트 | 카카오톡 조율 | 누락 월 8건, 불만 다수 | ★★★ |
| 성과 | 연 1회 정성평가 | 피드백 부재, 동기부여 저하 | ★★☆ |
| 급여 | 엑셀 수동계산 | 수당 11종, 오류 월 2-3건 | ★★★ |
| 교육 | 비체계적 OJT | 온보딩 14일, 품질 불균일 | ★★☆ |

### 3.2 근태관리 모듈

#### 3.2.1 출퇴근 체크 방식

| 방식 | 적용 대상 | 기술 | 비용 |
|------|----------|------|------|
| GPS 체크인 | 외근직 (배차/인수) | 모바일앱 위치 인증 반경 100m | 무료 (앱 내장) |
| QR 태그 | 사무직/매장 | 태블릿 QR 스캔 | 태블릿 1대 30만원 |
| 비콘 | 정비소 | BLE 비콘 자동 인식 | 비콘 1개 5만원 |

#### 3.2.2 근태 데이터 구조

```
Employee
├── employee_id (PK)
├── name, department, position
├── employment_type (정규/계약/파트)
├── base_salary, hourly_rate
└── AttendanceRecords[]
    ├── date
    ├── check_in (timestamp + GPS)
    ├── check_out (timestamp + GPS)
    ├── work_hours (auto-calc)
    ├── overtime_hours
    ├── late_flag (boolean)
    └── approval_status (대기/승인/반려)
```

#### 3.2.3 근태 규칙 엔진

| 규칙 | 조건 | 처리 |
|------|------|------|
| 지각 | 출근시간 +10분 초과 | 자동 플래그, 월 3회 시 알림 |
| 조퇴 | 퇴근시간 -30분 이전 | 사유 입력 필수, 반차 자동 차감 |
| 연장근무 | 일 8시간 초과 | 사전 승인 필요, 자동 수당 계산 |
| 야간근무 | 22:00-06:00 | 1.5배 수당 자동 적용 |
| 휴일근무 | 공휴일/일요일 | 2.0배 수당 자동 적용 |
| 무단결근 | 미체크인 + 무신고 | 즉시 팀장 알림, HR 에스컬레이션 |

### 3.3 시프트 관리 모듈

#### 3.3.1 시프트 유형

| 코드 | 시간 | 적용 | 최소인원 |
|------|------|------|---------|
| A (오픈) | 07:00-15:00 | 배차팀, 정비 | 4명 |
| B (미들) | 10:00-18:00 | CS, 사무 | 6명 |
| C (클로즈) | 14:00-22:00 | 반납, CS | 5명 |
| D (올나잇) | 22:00-07:00 | 비상 배차, 보안 | 2명 |
| F (플렉스) | 코어 10-16시 | 개발, 마케팅 | — |

#### 3.3.2 AI 시프트 최적화

**입력 변수**:
- 익일 예약 건수 (수요예측 모델 연동)
- 직원별 선호 시간대 (가중치 0.3)
- 연속 근무일수 (5일 초과 시 강제 휴무)
- 스킬 매칭 (차량 인수 가능 여부 등)

**최적화 목표**:
- 제약 1: 최소인원 충족
- 제약 2: 노동법 준수 (주 52시간)
- 제약 3: 연속 야간 2일 이하
- 목적함수: 인건비 최소화 + 직원 만족도 최대화 (Pareto 최적)

**알고리즘**: Constraint Programming (Google OR-Tools) + 직원 선호도 소프트 제약

#### 3.3.3 시프트 운영 플로우

```
매주 수요일 14:00
  → AI가 차주 시프트 초안 생성 (5분)
    → 팀장 검토/수정 (수요일 내)
      → 목요일 09:00 전직원 알림 (카카오톡 + 앱)
        → 금요일 18:00까지 변경 요청 가능
          → 월요일 확정 → 근태 시스템 자동 반영
```

### 3.4 성과관리 모듈

#### 3.4.1 OKR 체계

| 레벨 | Objective 예시 | Key Result 예시 | 주기 |
|------|---------------|----------------|------|
| 회사 | 제주 렌터카 1위 달성 | 월 매출 15억, NPS 72 | 분기 |
| 팀 | 배차 효율 극대화 | 평균 배차시간 12분, 가동률 78% | 분기 |
| 개인 | CS 품질 향상 | CSAT 4.5/5, 해결률 90% | 월간 |

#### 3.4.2 평가 프로세스

| 단계 | 시기 | 내용 | 도구 |
|------|------|------|------|
| 목표 설정 | 분기 초 (1주차) | 개인 OKR 3-5개 등록 | ERP 목표 탭 |
| 중간 체크인 | 분기 중 (6주차) | 1:1 면담, 진행률 업데이트 | 화상/대면 |
| 자기 평가 | 분기 말 (12주차) | KR 달성도 자기 입력 | ERP |
| 동료 평가 | 분기 말 (12주차) | 360도 피드백 (3-5인) | 익명 설문 |
| 최종 평가 | 분기 말 (13주차) | 팀장 종합 평가 + 등급 | A/B/C/D/F |
| 보상 연동 | 분기 말 (13주차) | 성과급 자동 산출 | 급여 모듈 |

#### 3.4.3 성과급 테이블

| 등급 | 비율 | 기본급 대비 | 추가 혜택 |
|------|------|-----------|----------|
| A (탁월) | 10% | +20% | 특별 휴가 2일, 교육비 지원 |
| B (우수) | 25% | +10% | 교육비 50만원 |
| C (양호) | 40% | 기본 | — |
| D (미흡) | 20% | -5% | 개선 계획 수립 필수 |
| F (부진) | 5% | -10% | PIP (성과개선프로그램) 진입 |

### 3.5 급여 자동화 모듈

#### 3.5.1 급여 구성 항목

| 분류 | 항목 | 산출 기준 | 자동화 |
|------|------|----------|--------|
| 기본급 | 월 기본급 | 직급/호봉 테이블 | 자동 |
| 고정수당 | 직책수당 | 직책별 고정액 | 자동 |
| 고정수당 | 식대 | 월 20만원 (비과세) | 자동 |
| 고정수당 | 교통비 | 월 10만원 (비과세) | 자동 |
| 변동수당 | 연장근로 | 통상시급 x 1.5 x 시간 | 근태 연동 |
| 변동수당 | 야간근로 | 통상시급 x 0.5 x 시간 | 근태 연동 |
| 변동수당 | 휴일근로 | 통상시급 x 1.5(or 2.0) x 시간 | 근태 연동 |
| 변동수당 | 성과급 | 등급별 비율 | 성과모듈 연동 |
| 공제 | 4대보험 | 요율 자동 적용 | 자동 |
| 공제 | 소득세/주민세 | 간이세액표 | 자동 |
| 공제 | 연차 미사용 정산 | 퇴직 시 or 연말 | 반자동 |

#### 3.5.2 급여 처리 타임라인

```
매월 20일: 근태 마감 → 자동 집계
매월 21일: 변동수당 자동 산출 → 팀장 확인
매월 22일: HR 최종 검증 (이상치 자동 하이라이트)
매월 23일: CFO 승인
매월 25일: 급여 이체 실행
매월 25일: 급여명세서 카카오 알림톡 발송
매월 말일: 4대보험/세금 신고 자동 연동
```

### 3.6 교육/온보딩 모듈 (LMS)

| 교육 트랙 | 대상 | 콘텐츠 수 | 필수 이수 기간 |
|----------|------|----------|--------------|
| 신입 온보딩 | 전 신입 | 12개 | 입사 후 7일 |
| 차량 지식 | 배차/정비 | 8개 | 입사 후 14일 |
| CS 스킬 | CS팀 | 6개 | 분기 1회 갱신 |
| 안전/법규 | 전직원 | 4개 | 연 1회 |
| 리더십 | 팀장급 | 5개 | 연 2회 |

**콘텐츠 형식**: 5분 내외 마이크로 러닝 영상 + 퀴즈 + 실습 과제
