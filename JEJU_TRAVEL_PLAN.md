> 버전: v0.1 | 최초 작성: 2026-05-16 | 상태: 초안

---

## 1. 프로젝트 개요

### 서비스 목적
"나에게 맞는 제주를 찾아드립니다."

제주 여행객은 정보 과잉 속에서 자신의 성향에 맞는 장소를 찾는 데 어려움을 겪는다. 블로그 상위 노출 맛집이 대기 1시간짜리임을 모르고 갔다가 실망하거나, 경치 좋은 카페를 찾다가 결국 유명 프랜차이즈에 앉는 상황이 반복된다. 본 서비스는 여행객의 성향(대기 감수도, 인파 감수도, 경치/맛/분위기 우선순위 등)을 수집하고, 실제 방문자 데이터와 결합해 "나에게 맞는 제주 장소"를 추천한다.

### 타겟 사용자
- 제주 방문 예정 여행객 (내국인 우선, 이후 외국인 확장)
- 연 1회 이상 제주를 방문하는 MZ 세대 (20~40대)
- 여행 계획을 직접 세우는 적극적 여행자

### 핵심 가치

| 가치 | 설명 |
|------|------|
| 성향 기반 | 같은 제주라도 사람마다 최적 장소가 다르다 |
| 데이터 신뢰 | 실제 방문자 코스 데이터 기반, 광고 없음 |
| 즉시 활용 | 추천 결과를 바로 코스로 변환 가능 |
| 커뮤니티 성장 | 방문자가 데이터를 기여하면 추천이 더 정확해짐 |

## 3. 사용자 페르소나

### 페르소나 A — "대기 감수형 미식가"
- **한마디**: "기다려도 좋으니까 제대로 된 맛집/빵집/카페 가고 싶어"
- **특징**: 줄 서는 것을 당연하게 여김. SNS에서 화제의 음식을 목적지로 설정. 이동 동선보다 맛을 우선시.
- **기피 요소**: 별로인 음식, 맛없는 카페
- **선호 카테고리**: 유명 맛집, 오픈런 빵집, 시그니처 메뉴 카페
- **성향 축 값**: 대기 감수도 상(60분) · 인파 감수도 중 · 맛 우선 · 계획형

---

### 페르소나 B — "인파 감수형 관광객"
- **한마디**: "사람 많아도 괜찮아, 유명하면 다 가볼래"
- **특징**: 성산일출봉, 협재해변 등 '제주의 대표 명소' 중심 여행. 여행 인증 사진 중시.
- **기피 요소**: 알려지지 않은 장소, 이동 거리 과다
- **선호 카테고리**: 관광지, 유명 해변, 드라마 촬영지
- **성향 축 값**: 대기 감수도 중 · 인파 감수도 상 · 경치 우선 · 계획형

---

### 페르소나 C — "경치 우선 힐링러"
- **한마디**: "대충 먹어도 되니까 경치만 예쁜 카페, 뷰 좋은 곳 데려다 줘"
- **특징**: 오션뷰/오름뷰 카페에서 시간 보내기 선호. 사람 없고 조용한 곳. 맛보다 분위기.
- **기피 요소**: 북적이는 장소, 긴 대기, 시내 맛집 탐방
- **선호 카테고리**: 뷰포인트 카페, 오름, 한적한 해변, 독립 카페
- **성향 축 값**: 대기 감수도 하(15분) · 인파 감수도 하 · 경치 최우선 · 즉흥형

---

### 페르소나 D — "효율 중시 가족 여행자"
- **한마디**: "애들이랑 오는데, 이동 적고 다양하게 즐길 수 있는 코스 주세요"
- **특징**: 어린 자녀 동반. 이동 시간 최소화, 주차 용이성 중요. 체험형 활동 선호.
- **기피 요소**: 긴 이동, 대기, 좁은 주차장, 미식 위주 코스
- **선호 카테고리**: 체험 프로그램, 넓은 관광지, 키즈 카페
- **성향 축 값**: 대기 감수도 하 · 인파 감수도 중 · 경치/맛 균형 · 계획형

## 4. 핵심 기능 명세

### 4-1. 성향 프로파일링

**목표**: 최대 90초 내 완료

| 축 | 질문 방식 | 선택지 |
|----|-----------|--------|
| 대기 감수도 | 슬라이더 | "지금 바로" ~ "1시간도 OK" |
| 인파 감수도 | 슬라이더 | "조용한 곳" ~ "북적이는 게 활기차" |
| 경치 vs 맛 | 카드 선택 3라운드 | 바다뷰 카페 vs 유명 디저트 카페 등 |
| 활동 강도 | 5점 척도 | 완전 휴식 ~ 하루 종일 액티비티 |
| 이동 방식 | 단일 선택 | 렌터카 / 대중교통 / 도보 위주 |
| 동행 유형 | 단일 선택 | 혼자 / 커플 / 가족 / 친구 / 효도 |

**산출물 타입**:
```typescript
interface PreferenceProfile {
  waitTolerance: number;       // 0~100
  crowdTolerance: number;      // 0~100
  sceneryVsFood: number;       // 0=경치, 100=맛
  activityLevel: number;       // 0~100
  transportMode: 'car' | 'transit' | 'walk';
  companion: 'solo' | 'couple' | 'family' | 'friends' | 'parents' | 'group';
}
```

---

### 4-2. 장소 추천 엔진

**Phase 1 — 룰 기반**:
- 각 장소에 수동 태그 부여: `대기시간(분)`, `혼잡도(1~5)`, `경치점수(1~5)`, `맛점수(1~5)`
- 사용자 성향 점수 × 장소 태그 가중치 → 매칭 점수 산출
- 카테고리별 상위 N개 반환

**Phase 3 — AI 유사도 기반**:
- 방문자 코스 데이터를 임베딩 처리 (OpenAI text-embedding-3-small)
- 유사 성향 사용자들이 방문한 장소 패턴으로 추천
- pgvector 벡터 유사도 검색

**추천 결과 구조**:
- 카테고리별 TOP 3 (맛집, 카페, 관광지, 뷰포인트)
- "성향이 비슷한 사람들이 자주 간 코스" 1개
- 지도 위 핀 시각화

---

### 4-3. 코스 저장 및 공유

- 추천 결과를 1-Day / 2-Day 코스로 자동 편성
- 사용자가 장소 추가/제거하여 커스터마이징
- 저장 시 고유 URL 생성 (비로그인 가능, 로그인 시 마이페이지 연동)
- 공유: URL 복사, 카카오톡 공유, OG 이미지 자동 생성
- 저장된 코스는 익명 처리 후 데이터 기여로 집계

---

### 4-4. 방문 후 데이터 수집 (크라우드소싱)

여행 후 간단한 리뷰 플로우 (최대 5탭):
1. 실제로 간 장소 체크 (추천 목록에서 선택)
2. 대기시간 실측 입력
3. 혼잡도 체감 (1~5)
4. 만족도 (엄지 업/다운)
5. 한줄 코멘트 (선택)

기여 보상: 다음 여행 시 추천 정확도 향상 배지, 포인트 (Phase 2)

---

### 4-5. 관리자 대시보드

- 장소별 방문자 집계 및 성향 분포 시각화
- 태그 레이블 수정 UI
- 추천 엔진 성능 지표 (클릭률, 코스 저장률)
- 데이터 품질 알림 (이상값 자동 감지)

## 5. 데이터 전략

### 5-1. 초기 씨드 데이터 (Phase 1, 출시 전 4주)

**수집 목표**: 맛집 300개 + 카페 200개 + 관광지 100개 + 뷰포인트 50개 = 총 650개 장소

**수집 방법**:

| 소스 | 수집 내용 | 비용 |
|------|-----------|------|
| 한국관광공사 TourAPI | 기본 정보, 위치, 이미지 | 무료 |
| 네이버 플레이스 API | 별점, 리뷰 수, 대표 키워드 | 월 25만건 무료 |
| 카카오맵 API | 혼잡도, 영업시간 | 월 30만건 무료 |
| 수동 큐레이션 | 성향 태그 레이블링 | 인건비 |

**태그 레이블링 기준**:

| 태그 | 측정 방법 |
|------|-----------|
| 평균 대기시간(분) | 네이버 리뷰 텍스트 분석 + 수동 |
| 혼잡도 (1~5) | 구글맵 인기 시간대 데이터 |
| 경치 점수 (1~5) | 인스타그램 해시태그 분석 + 수동 |
| 맛 점수 (1~5) | 네이버 별점 + 리뷰 키워드 |
| 주차 난이도 | 수동 (쉬움/보통/어려움) |
| 어린이 친화 | 리뷰 키워드 분석 |

---

### 5-2. 크라우드소싱 전환 (Phase 2)

- 방문자가 대기시간, 혼잡도를 입력하면 씨드 데이터를 점진적으로 대체
- 기여 데이터 5건 이상 수집 장소에 "검증됨" 배지 부여
- 어뷰징 방지: 같은 사용자 동일 장소 30일 재입력 제한, 이상값 자동 격리

---

### 5-3. 외부 API 활용 계획

| API | 용도 | 비용 |
|-----|------|------|
| 한국관광공사 TourAPI | 장소 기본 정보 | 무료 |

## 6. UX/UI 디자인 방향

### 6-1. 온보딩 플로우

```
[시작 화면]
  ↓ "제주 여행 성향 찾기" CTA
[Step 1] 동행 유형 선택 (카드 6개)
  ↓
[Step 2] 이동 방식 선택
  ↓
[Step 3~4] 성향 슬라이더 (2개씩 2화면)
  ↓
[Step 5] "이것 vs 저것" 카드 선택 3라운드
  ↓
[결과] 성향 유형 카드 + 추천 장소 즉시 표시
```

**설계 원칙**:
- 진행 상태 표시 (5단계 중 N단계)
- 중간 이탈 시 로컬스토리지 임시 저장
- 처음부터 다시 하기 버튼 상시 노출

---

### 6-2. 성향 입력 UI 컨셉

**슬라이더 UI**:
- 왼쪽/오른쪽 극단 레이블 표시 (예: "바로 들어가고 싶어" ↔ "1시간도 기다려")
- 슬라이더 이동 시 실시간으로 대표 장소 예시 이미지 변경

**카드 선택 UI**:
- 두 장소 카드를 좌우 배치
- 각 카드: 대표 이미지 + 장소 특징 3개
- 탭 시 선택 카드 확대, 비선택 페이드 애니메이션
- 3라운드 후 "성향 분석 중..." 로딩 (0.8초)

---

### 6-3. 추천 결과 화면 구조

```
[헤더]
  성향 유형 배지 (예: "경치 우선 힐링러")
  성향 요약 2줄

[섹션 1] 카테고리별 추천 장소 (가로 스크롤 카드)
  ├ 맛집 TOP 3
  ├ 카페 TOP 3
  ├ 관광지 TOP 3
  └ 뷰포인트 TOP 3

[섹션 2] 나 같은 여행자의 코스
  └ 유사 성향 사용자 인기 코스 1개 (타임라인)

[섹션 3] 지도 보기 (카카오맵)

[하단 CTA]
  ├ "이 장소로 코스 만들기"
  └ "성향 다시 설정"
```

**장소 카드 컴포넌트**:
- 이미지 (3:2 비율)
- 장소명 + 카테고리 배지
- 매칭 이유 한줄 (예: "대기 15분 이내 · 오션뷰")
- 성향 일치율 (예: "당신과 87% 일치")
- 찜하기 / 코스에 추가 버튼

| 항목 | 선택 |
|------|------|

| DB | Supabase (PostgreSQL) |

### DB 주요 테이블

```sql
-- 장소 마스터
places (
  id UUID PRIMARY KEY,
  name TEXT,
  category TEXT,  -- cafe|restaurant|attraction|viewpoint|beach|trail
  lat FLOAT, lng FLOAT,
  address TEXT, image_url TEXT,
  avg_wait_minutes INT,
  crowd_score FLOAT,    -- 1~5
  scenery_score FLOAT,  -- 1~5
  food_score FLOAT,     -- 1~5
  parking_ease TEXT,    -- easy|medium|hard
  child_friendly BOOLEAN,
  embedding VECTOR(1536),  -- Phase 3
  verified BOOLEAN DEFAULT FALSE
)

-- 성향 프로필
user_preferences (
  id UUID PRIMARY KEY,
  user_id UUID,
  session_id TEXT,
  wait_tolerance INT,
  crowd_tolerance INT,
  scenery_vs_food INT,
  activity_level INT,
  transport_mode TEXT,
  companion TEXT,
  created_at TIMESTAMPTZ
)

-- 저장된 코스
saved_courses (
  id UUID PRIMARY KEY,
  user_id UUID,
  preference_id UUID,
  name TEXT, days INT,
  spots JSONB,
  share_token TEXT UNIQUE,
  is_public BOOLEAN DEFAULT TRUE,
  created_at TIMESTAMPTZ
)

-- 방문 후 기여
place_reviews (
  id UUID PRIMARY KEY,
  place_id UUID,
  user_id UUID,
  actual_wait_minutes INT,
  crowd_rating INT,
  satisfaction BOOLEAN,
  comment TEXT,
  visited_at DATE
)
```
